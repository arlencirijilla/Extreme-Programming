Extreme Programming Explained

Embrace Change

XP is

  • Lightweight
  • A methodology based on addressing constraints of Software Development
  • Scalable although larger teams augment practices
  • Adaptable to vague or rapidly changing requirements

Values

  • Communication
  • Feedback
  • Simplicity
  • Courage
  • Respect

Practices

  • Aim is to improve Software Development
  • Complement each other
  • Express the values
